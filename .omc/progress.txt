### US-001: 변경사항 요약

#### camera_service.dart
- `_Now` typedef 추가 (`DateTime Function()`) — 시간 의존성 주입 가능
- `CameraService.testable()` 팩토리 생성자 추가 (`@visibleForTesting`)
  - `requestGalleryPermission`, `getAssetPathList`, `now` 세 의존성 모두 주입 가능
- `_firstGalleryImageInFlight: Future<AssetEntity?>?` — in-flight dedupe 추가
- `_galleryPermissionInFlight: Future<PermissionState>?` — permission in-flight dedupe 추가
- `getGalleryPermissionState()` 공개 메서드 추가 (TTL + in-flight dedupe)
- `_requestGalleryPermissionState()` 내부 메서드 분리
- `_invalidateGalleryCache()` 추가 — takePicture/stopVideoRecording 성공 후 호출
- `getFirstGalleryImage()` 공개 메서드 추가 (TTL 5s + in-flight dedupe)
- `_loadFirstGalleryImageFromGallery()` 내부 메서드 분리

#### gallery_thumbnail.dart
- `_thumbnailFuture: Future<Uint8List?>?` — Future 인스턴스 캐싱
- `_thumbnailAssetId: String?` — 마지막 렌더링 asset ID 추적
- `_thumbnailSize: double?` — 마지막 렌더링 size 추적
- `_syncThumbnailFuture()` 메서드 추가 — asset.id + size 비교, 변경 시에만 새 Future 생성
- initState/didUpdateWidget 양쪽에서 `_syncThumbnailFuture()` 호출
- 결과: 동일 asset+size 리빌드 시 `thumbnailDataWithSize` 중복 호출 방지

#### camera_screen.dart
- `_firstGalleryImage: AssetEntity?` — 이전 String path 방식에서 AssetEntity 직접 보유로 변경
- `GalleryThumbnail` 위젯에 AssetEntity 직접 전달
- `_loadFirstGalleryImage()`: `CameraService.getFirstGalleryImage()` 사용 (캐시 활용)
- iOS 재시도 로직: 800ms + 2000ms delay retry (Photo Library 준비 대기)

### US-004: 개선 필요 항목 (우선순위별)

#### [높음] 미커버 테스트 케이스
1. `_invalidateGalleryCache()` 동작 검증 없음
   - takePicture/stopVideoRecording 호출 후 캐시가 실제로 무효화되어 다음 getFirstGalleryImage()가 새 값을 반환하는지 미검증
2. `getGalleryPermissionState()` in-flight dedupe 미검증
   - permission 레이어의 동시 호출 dedupe는 getFirstGalleryImage in-flight 테스트로 간접 검증만 됨
3. GalleryThumbnail — size만 변경 시 새 Future 생성 미검증

#### [중간] 코드 품질 이슈
4. `optimizeCamera()` 빈 catch 블록 (camera_service.dart:522-523)
   ```dart
   if (e.code == 'unimplemented') {
   } else {} // 아무것도 안 함
   ```
   → else 브랜치 제거 또는 debugPrint 추가 권장

5. `_isRecording: bool = false` 필드가 항상 false (camera_service.dart:119)
   — `final` + 초기값 고정이므로 사실상 미사용 dead code

#### [낮음] 구조적 관찰
6. `getFirstGalleryImage()`와 `loadLatestGalleryImage()`가 병렬 존재
   - getFirstGalleryImage: AssetEntity 반환, TTL 5s 캐시 (신규)
   - loadLatestGalleryImage: String path를 _latestGalleryImagePath에 저장 (구형)
   - camera_screen.dart는 신규 방식만 사용하므로 구형 메서드 제거 검토 가능

7. iOS 재시도 로직의 매직 넘버 (800ms, 2000ms) — 상수화 권장
